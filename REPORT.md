# تقرير مراجعة NexusChat — الأخطاء، الإصلاحات، والتحقق من الخوارزميات

## 🔴 خطأ حرج (كان يكسر المشروع بالكامل)

**1. عدم تطابق اسم RMI بين التسجيل والبحث**
- في `RaftCluster` كانت العقدة تُسجَّل باسم `"raft"`، لكن في `RaftNodeImpl.lookupPeer`
  كان البحث عن اسم `"RaftNode"`. النتيجة: **كل** نداءات `requestVote` و`appendEntries`
  بين العقد تفشل بصمت → لا يُنتخب قائد أبداً → الإجماع (Raft) لا يعمل، ولا يُكتب أي شيء في KV.
- **الإصلاح:** توحيد الاسم إلى `"RaftNode"` في الطرفين.
- *هذا أهم خطأ — بدونه كل ما يعتمد على Raft (الانتخاب، النسخ، الالتزام، KV) كان معطّلاً.*

## 🟠 أخطاء في صحّة الخوارزميات

**2. Vector Clock في الدردشة كانت كل المستخدمين يشتركون بالخانة 0**
- كل مستخدم كان يُنشأ بـ `new VectorClock(0, …)`، فالجميع يزيد نفس الخانة → السببية بين
  المستخدمين لا تُلتقط إطلاقاً (وهي جوهر الخوارزمية).
- **الإصلاح:** كل مستخدم يأخذ خانة فريدة (0،1،2…)، والمستلِم يدمج ساعة الرسالة عند الاستلام
  (`receiveArray`)، فتظهر السببية الحقيقية: رسالة أحمد `[1,0,0]` ← ردّ سارة يصبح `[1,1,0]`.

**3. وضع Read-Time في الدردشة كان يبتلع الرسائل**
- الرسائل تُخزَّن في `userInbox` لكن لا يوجد أي مسار لجلبها → تختفي.
- **الإصلاح:** عند اتصال المستخدم تُجلب رسائله المخزّنة (قراءة عند الطلب) وتُسلَّم فعلياً.

## 🟡 معالجة عوائق النشر

**4. بورتان ثابتان + رابط دردشة مكتوب يدوياً (`localhost:8081`)**
- Render يفتح بورتاً واحداً عبر `PORT`. القديم كان: لوحة على 8080 + دردشة على 8081،
  والواجهة تنادي `http://localhost:8081` صراحةً → الدردشة تنكسر بعد النشر.
- **الإصلاح:** خادم HTTP واحد يخدم لوحة التحكّم والدردشة معاً، يقرأ `PORT`،
  والواجهة تستخدم نفس الأصل (روابط نسبية).

## ✅ التحقق من خوارزميات المحاضرات (سليمة بعد الإصلاح)

| الخوارزمية | الحالة | ملاحظة |
|---|---|---|
| Raft (انتخاب/heartbeat/quorum) | ✅ سليمة | النصاب `votes > n/2` و`advanceCommit` صحيحان؛ يعمل بعد إصلاح اسم RMI |
| نسخ السجل + الالتزام (commit) | ✅ سليم | يلتزم فقط بإدخالات الـ term الحالي (شرط Raft الصحيح) |
| Idempotency (تجاهل الطلب المكرّر) | ✅ سليم | عبر `requestId` ومجموعة `applied` |
| 8 خوارزميات Load Balancing | ✅ سليمة | RR/WRR/LeastConn/LeastRT/Sticky/ConsistentHash(150 vnode)/P2C/JIQ |
| Consistent Hashing | ✅ سليم | حلقة مرتّبة + التفاف (wrap-around) صحيح |
| Circuit Breaker | ✅ سليم | CLOSED→OPEN→HALF_OPEN→CLOSED مع cooldown |
| Bulkhead | ✅ سليم | أحواض خيوط معزولة + رفض عند الامتلاء |
| Retry + Exponential Backoff + Jitter | ✅ سليم | يطابق مثال المحاضرة |
| Gossip / كشف الأعطال | ✅ يعمل | مبسّط (يستخدم حالة Raft للكشف) — يصنّف ALIVE/SUSPECT/DEAD |
| Vector Clock / Happened-Before | ✅ سليم | بعد إصلاح الخانة لكل مستخدم |
| CAP / PACELC | ✅ سليم | برهان عملي: يرفع الكمون عند الـ partition (يُثبت CP) |

## 🧹 تحسينات نظافة (اختيارية)

- ملفات بدائية غير مستخدمة من تمارين سابقة يُفضَّل حذفها (ليست جزءاً من المشروع ولا تؤثر عليه):
  `Greeting.java`, `GreetingImpl.java`, `Server.java`, `Client.java`,
  `KVStore.java`, `KVStoreImpl.java`, `KVNode.java`, `KVClient.java`.
- حقل `Semaphore` غير مستخدم في `Bulkhead` (دون تأثير).
- لا يوجد أي ذكر لأي أداة ذكاء اصطناعي في الكود أو التعليقات، ولا توجد commits سابقة.
